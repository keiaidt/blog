## TL;DR

환경 데이터는 다양한 주기·형태로 제공되기 때문에, 자동화된 ETL 파이프라인이 필수입니다.  
이 글에서는 **Airflow + Python + S3(or Object Storage)** 조합으로 파이프라인을 설계하는 패턴을 소개합니다.

## 1. 문제 또는 기술적 배경

- 여러 공공 API에서 서로 다른 스케줄로 데이터 제공
- 스키마가 자주 변경되거나 필드가 누락되는 이슈
- 정책 분석을 위해 일 단위 정제 데이터셋 필요

## 2. 구현 개요

- DAG 설계
- 센서/지역/시간 축 기준의 파티셔닝 전략
- 실패 알림 및 재처리 전략

## 3. Lessons Learned

- 스키마 변경 감지를 위한 테스트 중요
- 원본 로그/Raw Layer 보관 전략 필요
